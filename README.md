# tools-setup
`tools-setup` 是一个专为 **Bug Bounty（漏洞赏金）** 和 **红蓝对抗** 设计的自动化工具部署方案。它的核心理念是“**即插即用**”：通过 Bash 脚本直接从 GitHub Release 获取预编译的二进制文件，彻底告别复杂的开发环境配置。

---

## 核心特性

*   **零依赖环境**：不需要预装 Go、Rust、Python 或 GCC 编译环境，避免了因依赖冲突导致的“环境地狱”。
*   **智能架构适配**：脚本内置双重过滤逻辑（白名单 + 黑名单），能自动识别 `amd64` / `x86_64` 架构，并精准处理不规范命名的仓库。
*   **动态更新机制**：
    *   默认模式：仅安装本地缺失的工具。
    *   强制模式 (`-f`)：一键比对并覆盖安装 GitHub 上的最新版本。
*   **格式支持广泛**：完美适配 `.zip`、`.tar.gz`、`.tgz` 等压缩包格式以及单可执行文件。

---

## 逻辑架构

脚本的运行流程非常清晰，保证了在 VPS 或新服务器上部署的极速体验：

1.  **环境检查**：确认 root 权限，安装必要的 `jq`、`curl` 等基础解析工具。
2.  **资产检索**：调用 GitHub API 获取最新的 Release 元数据。
3.  **多级匹配**：
    *   优先匹配标注了 `linux` 且含有 `x64` 关键词的包。
    *   若失败，则启动黑名单过滤，排除 `win`、`mac`、`arm`、`i386` 等干扰项。
4.  **智能安装**：可执行文件统一放至 `/usr/local/bin` 目录下。

脚本安装的工具包括但不限于：

| 类别 | 代表工具 |
| :--- | :--- |
| **子域/资产发现** | `subfinder`, `findomain`, `assetfinder` |
| **端口与指纹识别** | `naabu`, `httpx`, `asnmap` |
| **漏洞扫描 (PoC)** | `nuclei`, `dalfox`, `jaeles` |
| **爬虫与路径提取** | `katana`,  `waybackurls`, `gau` |
| **参数/隐藏挖掘** | `ffuf`, `x8`, `gf` |
